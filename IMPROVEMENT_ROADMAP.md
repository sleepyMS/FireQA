## Phase 1: 긴급 — 기반 품질 확보 (1~2주)

> 프로덕션 리스크 제거 및 개발 안정성 확보

### 1.1 API 입력 검증에 Zod 도입

- **현재 문제**: 74개 API 라우트에서 `request.json()` 결과를 타입 단언(`as`)으로만 처리. 입력 검증 거의 없음
- **위험**: 인젝션, 예상치 못한 타입 에러, 런타임 크래시
- **개선**: Zod 스키마로 모든 API 입력 검증. `src/lib/api/schemas/` 디렉토리에 도메인별 스키마 정의
- **영향 범위**: `src/app/api/` 전체 (74개 라우트)
- **우선순위**: 높음
- **참조**: `TECHNICAL_ANALYSIS.md` > 1. API 라우트 패턴 일관성

### 1.2 중앙 에러 처리 시스템

- **현재 문제**: 모든 에러가 `status: 500`으로 반환. 사용자 입력 오류(400)와 시스템 오류(500) 미분화. 에러 원인이 클라이언트에 전달되지 않음
- **개선**: `ApiError` 클래스 + 중앙 에러 핸들러 도입. Prisma 에러 맵핑 (`P2025` → 404 등)
- **영향 범위**: `src/app/api/` 전체, `src/lib/api/` 신규
- **우선순위**: 높음
- **참조**: `TECHNICAL_ANALYSIS.md` > 1.1 일관되지 않은 에러 처리

### 1.3 타입 안전성 강화

- **현재 문제**: `as` 타입 단언 남용, `JSON.parse` 결과 미검증
- **개선**: 타입 단언 제거 → Zod 파싱으로 대체. 런타임 타입 가드 적용
- **영향 범위**: `src/app/api/`, `src/lib/`
- **우선순위**: 높음
- **참조**: `TECHNICAL_ANALYSIS.md` > 3. 타입 안전성

### 1.4 에러 페이지 개선

- **현재 문제**: error.tsx, not-found.tsx의 사용자 피드백이 부족하거나 빈약
- **개선**: 사용자 친화적 에러 페이지 (홈으로 이동, 새로고침, 오류 신고 등 액션 제공)
- **영향 범위**: `src/app/` 에러 바운더리 파일들
- **우선순위**: 높음
- **참조**: `UX_UI_IMPROVEMENT_REPORT.md` > 5. 에러 처리 UX

## Phase 2: 핵심 기능 보강 (2~4주)

> 사용자 실무 도입을 위한 필수 기능 확보

### 2.1 TC 직접 편집 기능

- **현재 상태**: 생성된 테스트케이스가 읽기 전용. 다이어그램은 Mermaid 코드 편집 가능하나 TC/와이어프레임은 편집 불가
- **개선**: TC 항목별 인라인 편집 UI. 편집 시 새 버전 자동 생성. 일괄 편집(bulk edit) 지원
- **비즈니스 임팩트**: 높음 — 생성 결과를 수정할 수 없으면 실무 도입 불가
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 1. 생성 기능 완성도 > 편집 기능

### 2.2 공개 링크 공유

- **현재 상태**: 생성 결과를 팀 외부인에게 공유할 방법 없음. 내보내기(JSON/Excel/Markdown/Mermaid)만 가능
- **개선**: 읽기 전용 공개 링크 생성. 만료 기한/비밀번호 설정 옵션. 공유 링크 관리 UI
- **비즈니스 임팩트**: 높음 — 협업 SaaS의 필수 기능
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 2. 내보내기/공유 기능

### 2.3 API 공통 미들웨어 추출

- **현재 문제**: 조직 권한 검증, 데이터 변환(toISOString), 제한 정책 로직이 각 라우트마다 중복
- **개선**: 공통 미들웨어/헬퍼 함수로 추출. `withAuth()`, `withOrgAccess()`, `withValidation()` 래퍼
- **영향 범위**: `src/app/api/` 전체, `src/lib/api/` 신규
- **우선순위**: 중간
- **참조**: `TECHNICAL_ANALYSIS.md` > 6. 코드 중복

### 2.4 테스트 기초 구축

- **현재 상태**: 테스트 파일 1개만 존재. 74개 API 라우트 중 0개 테스트
- **개선**: Vitest 기반 API 통합 테스트. 핵심 라우트(생성, 인증, 과금) 우선. 목표: 핵심 API 70%+ 커버리지
- **우선순위**: 중간
- **참조**: `TECHNICAL_ANALYSIS.md` > 4. 테스트 커버리지

### 2.5 SSE 스트리밍 진행률 개선

- **현재 문제**: 생성 중 구체적 진행 상황이 부족. 어떤 단계인지, 얼마나 남았는지 불명확
- **개선**: 단계별 진행률 표시 (문서 파싱 → 프롬프트 생성 → AI 호출 → 결과 처리). 예상 소요 시간 표시
- **우선순위**: 중간
- **참조**: `UX_UI_IMPROVEMENT_REPORT.md` > 3. 생성 결과 UX

## Phase 3: QA SaaS 차별화 (1~2개월)

> QA 도구로서의 핵심 가치 제공 및 성능 최적화

### 3.1 테스트 실행/추적 시스템

- **현재 상태**: TC 생성만 가능. 생성된 TC의 실행 상태(통과/실패/스킵) 관리 불가
- **개선**: TC 실행 상태 관리 (pass/fail/skip/blocked). 테스트 실행 기록(test run) 개념 도입. 실행 결과 대시보드 (통과율, 실패 항목 등)
- **비즈니스 임팩트**: 매우 높음 — QA SaaS의 핵심 가치. 생성→실행→추적→회귀의 완전한 QA 라이프사이클
- **구현 복잡도**: 높음 (새 데이터 모델, UI, API 필요)
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 10. QA 특화 기능

### 3.2 Jira/Slack 통합

- **현재 상태**: 외부 도구 연동 없음 (Figma 플러그인만 존재)
- **개선**: Jira 연동 (TC→이슈 동기화, 실패 TC→버그 자동 생성). Slack 알림 (생성 완료, 댓글, 테스트 결과). Webhook 지원 (커스텀 통합)
- **비즈니스 임팩트**: 높음 — QA 팀의 실제 워크플로우와 연결
- **구현 복잡도**: 중간~높음
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 8. 통합 기능

### 3.3 버전 간 비교 UI

- **현재 상태**: 버전 관리 시스템은 완전히 구현됨 (ResultVersion, DiagramVersion). 그러나 두 버전의 차이를 시각화하는 기능 없음
- **개선**: 버전 diff 비교 UI (텍스트 diff, 시각적 diff). 롤백 UI. 변경 히스토리 타임라인
- **비즈니스 임팩트**: 중간
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 1. 생성 기능 완성도 > 버전 비교

### 3.4 SSE 폴링 최적화

- **현재 문제**: SSE 폴링이 1초마다 DB를 조회. 수백 사용자 동시 접속 시 심각한 DB 부하
- **개선**: 폴링 간격 동적 조정 (작업 초반 1초 → 점차 늘림). 또는 PostgreSQL LISTEN/NOTIFY 기반 이벤트 드리븐으로 전환
- **영향 범위**: `src/app/api/` SSE 관련 라우트
- **우선순위**: 중간
- **참조**: `TECHNICAL_ANALYSIS.md` > 5. 성능 이슈

### 3.5 N+1 쿼리 최적화

- **현재 문제**: Prisma 쿼리에서 N+1 쿼리 위험 존재
- **개선**: Prisma `include`/`select` 최적화. 불필요한 관계 로딩 제거. 필요시 raw query 사용
- **영향 범위**: `src/app/api/` 전체
- **우선순위**: 중간
- **참조**: `TECHNICAL_ANALYSIS.md` > 5. 성능 이슈 > N+1

### 3.6 구조화된 로깅 도입

- **현재 문제**: `console.error`만 사용. 구조화된 로깅 없음. 에러 추적(Sentry 등) 미연동
- **개선**: JSON 형식 구조화 로깅. 요청 ID 기반 추적. Sentry 또는 유사 서비스 연동
- **우선순위**: 중간
- **참조**: `TECHNICAL_ANALYSIS.md` > 7. 로깅/모니터링

## Phase 4: 사용성 고도화 (2~4개월)

> UX 완성도 및 고급 기능 확장

### 4.1 모바일 네비게이션 재설계

- **현재 문제**: 1024px(lg) 이상에서만 사이드바 최적화. 모바일에서 네비게이션 사용 불편
- **개선**: 모바일 전용 네비게이션 (바텀 네비게이션 또는 햄버거 메뉴). 터치 영역 최소 44px 보장. 주요 페이지의 모바일 레이아웃 최적화
- **우선순위**: 중간
- **참조**: `UX_UI_IMPROVEMENT_REPORT.md` > 2. 네비게이션, 4. 반응형 디자인

### 4.2 접근성(a11y) 강화

- **현재 문제**: ARIA 속성 부분적, 키보드 네비게이션 미흡, 색상 대비 일부 불충분
- **개선**: ARIA 속성 체계적 적용. 키보드 네비게이션 완전 지원. WCAG 2.1 AA 기준 색상 대비 충족
- **우선순위**: 중간
- **참조**: `UX_UI_IMPROVEMENT_REPORT.md` > 6. 접근성

### 4.3 AI 모델 선택

- **현재 상태**: OpenAI(GPT) 단일 모델만 지원
- **개선**: Claude, Gemini 등 추가 AI 모델 선택 옵션. 모델별 품질/속도/비용 비교 안내. 프로젝트별 기본 모델 설정
- **비즈니스 임팩트**: 중간 — 사용자 선택권 및 비용 최적화
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 9. AI 기능 고도화

### 4.4 커스텀 프롬프트/템플릿

- **현재 상태**: QATemplate 모델은 존재하나 사용자 커스텀 프롬프트 기능 제한적
- **개선**: 도메인별 맞춤 프롬프트 작성 UI. 팀 내 템플릿 공유. 프롬프트 마켓플레이스
- **비즈니스 임팩트**: 중간 — 도메인 특화 QA 지원
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 9. AI 기능 고도화

### 4.5 내보내기 확장

- **현재 지원**: JSON, Excel, Markdown, Mermaid (4가지)
- **추가 필요**: PDF (경영진/기획자용), Word/DOCX (MS Office 호환), CSV (데이터 분석용), Figma 양방향 동기화

### 4.6 협업 고도화

- **현재 상태**: 댓글+답글, 인앱 알림, 역할 기반 접근 제어는 구현됨
- **추가 필요**: @멘션 기능, 실시간 동시 편집, 이메일/푸시 알림, Slack 알림 통합

- **구현 복잡도**: 중간~높음

### 4.7 어드민 패널

- **현재 상태**: 관리자용 UI 없음. 사용자/조직/시스템 관리 불가
- **개선**: 어드민 대시보드 (사용자 관리, 조직 관리, 사용량 통계, 시스템 상태). 슈퍼어드민 역할 추가
- **비즈니스 임팩트**: 중간 — 운영 필수
- **구현 복잡도**: 중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 7. 관리자 기능

### 4.8 사용량 대시보드 + 크레딧 잔액 알림

- **현재 상태**: 크레딧 시스템은 구현되어 있으나 사용자에게 사용량 시각화/알림 부재
- **개선**: 크레딧 사용량 그래프, 잔액 알림 (임계치 도달 시), 인보이스/영수증 다운로드
- **비즈니스 임팩트**: 중간 — 과금 투명성
- **구현 복잡도**: 낮음~중간
- **참조**: `FUNCTION_COMPLETENESS_ANALYSIS.md` > 6. 과금/빌링 완성도

### 4.9 로딩/빈 상태 UX 개선

- **현재 문제**: 스켈레톤 UI 부분적 적용. 빈 상태(프로젝트 없음, 결과 없음 등) 처리가 일관적이지 않음
- **개선**: 전체 페이지에 스켈레톤/Suspense 일관 적용. 빈 상태에 CTA(Call-to-Action) 포함한 일러스트레이션
- **우선순위**: 낮음
- **참조**: `UX_UI_IMPROVEMENT_REPORT.md` > 7. 빈 상태, 8. 로딩 상태

## 참고: 현재 잘 구현된 부분

개선에 집중하되, 현재 잘 되어 있는 부분도 인지해야 한다:

- **4가지 생성 기능 (TC, 다이어그램, 와이어프레임, 기획서 개선)**: 완전히 구현됨
- **SSE 실시간 스트리밍**: 생성 진행 상황 실시간 전달
- **멀티테넌시 + RBAC**: 조직 격리, 역할 기반 접근 제어
- **Stripe 과금 + 크레딧 시스템**: 원자적 트랜잭션 기반 안전한 과금
- **에이전트 시스템 (Phase 4)**: Hosted Worker + 자동 스케일링
- **버전 관리**: 자동/수동 버전 생성, 활성화/비활성화
- **다크모드 + 다국어 (ko/en)**: 기본 지원
- **Figma 플러그인**: 다이어그램/와이어프레임 자동 배치
