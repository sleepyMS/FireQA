# FireQA 기능 완성도 분석 리포트

**분석 일시**: 2026-04-04  
**분석 범위**: 코드 기반 기능 완성도 및 확장 가능성 평가

---

## 1. 생성 기능 완성도

### 현재 구현 상태

#### 테스트케이스(TC) 생성 ✅
- **구현**: 완전히 구현됨
- **코드 근거**: `src/app/api/generate/route.ts` (POST), `src/lib/openai/prompts/test-case-system.ts`
- **기능**:
  - PDF/Excel/Word 문서 파싱 후 TC 생성
  - 청크 단위 스트리밍 생성 (SSE)
  - 템플릿 기반 커스터마이징 (`QATemplate` 모델)
  - 청크 기반 분할 처리 (`splitDocument`)

#### 다이어그램 생성 ✅
- **구현**: 완전히 구현됨
- **코드 근거**: `src/app/api/diagrams/route.ts`, `src/lib/openai/schemas/diagram.ts`
- **기능**:
  - Mermaid 코드 생성
  - 노드/엣지 정보 저장 (`nodesJson`, `edgesJson`)
  - 버전 관리 (`DiagramVersion` 모델)

#### 와이어프레임 생성 ✅
- **구현**: 완전히 구현됨
- **코드 근거**: `src/app/api/wireframes/route.ts`, `src/lib/openai/schemas/wireframe.ts`
- **기능**:
  - 스크린 기반 와이프레임 생성

#### 기획서 개선 ✅
- **구현**: 완전히 구현됨
- **코드 근거**: `src/app/api/improve/route.ts`, `src/lib/openai/prompts/spec-improve-system.ts`
- **기능**:
  - 마크다운 형식 개선안 생성
  - Markdown 내보내기 지원

### 편집/수정 기능

#### 현재 상태: **부분적**
- **있음**:
  - Mermaid 코드 직접 편집 (Diagram 페이지)
  - AI 개선 API (`/api/improve-diagram`, `/api/fix-mermaid`)
- **부족한 점**:
  - **TC 직접 편집 기능 부재**: UI에서 생성된 TC 수정 불가 (읽기 전용)
  - **와이어프레임 편집 기능 부재**: 생성 후 수정 불가
  - **일괄 편집 기능 부재**: 여러 항목 동시 수정 불가

### 버전 관리

#### 현재 상태: **완전히 구현됨** ✅
- **코드 근거**: `prisma/schema.prisma` (ResultVersion, DiagramVersion)
- **기능**:
  - 자동 버전 생성 (생성 시 v1)
  - 수동 버전 생성 API (`POST /api/versions`)
  - 버전 활성화/비활성화 (`PUT /api/versions/[id]/activate`)
  - 변경 유형 추적 (initial, ai-improve, ai-fix, manual-edit, revert)
  - 변경 요약 및 명령어 기록

#### 개선 영역:
- **버전 간 비교 기능 부재**: 두 버전의 차이를 시각화하는 기능 없음
- **버전 롤백 UI 부재**: 이전 버전으로 되돌리기 UI가 없음
- **변경 히스토리 UI 부재**: 어떤 변경이 언제 누가 했는지 보여주는 UI 부족

### 생성 결과 비교

#### 현재 상태: **없음** ❌
- **문제**: 같은 프로젝트에서 여러 번 생성한 결과를 비교할 수 없음
- **영향**: "이전 버전이 나았는가?" 판단 어려움

## 2. 내보내기/공유 기능

### 내보내기 포맷

#### 현재 구현: **4가지 포맷**
- **JSON** (`/api/export/json`): 원본 생성 결과 (모든 메타데이터 포함)
- **Excel** (`/api/export/excel`): TC 전용, 구조화된 스프레드시트
- **Markdown** (`/api/export/markdown`): 기획서 개선 결과만
- **Mermaid** (`/api/export/mermaid`): 다이어그램 원본 코드

#### 부족한 포맷:
- **❌ PDF**: 프리젠테이션 형식 PDF 내보내기 없음 (일반 기획자/경영진용)
- **❌ Word/DOCX**: Microsoft Office 호환성
- **❌ Figma**: 와이어프레임을 Figma로 직접 전송 (현재 플러그인은 토큰만 관리)
- **❌ CSV**: TC 간단한 형식
- **❌ HTML**: 웹 기반 공유

### 공개 링크/외부 공유

#### 현재 상태: **없음** ❌
- **문제**: 생성 결과를 팀 외부인에게 공유할 수 없음 (로그인 필수)
- **영향**:
  - 스테이크홀더 리뷰 불가능
  - 고객에게 결과물 전달 불가능
  - 외부 컨설턴트와의 협업 제한

### 파일 생성 내역 추적

#### 현재 상태: **부분적**
- **있음**: 생성 날짜, 생성자 정보
- **부족함**: 내보내기 이력 추적 안 됨

### 비즈니스 임팩트
- **높음**: 공개 링크/외부 공유는 SaaS 기본 요구사항 (협업 가치 손실)
- **높음**: PDF 내보내기는 기획자/경영진이 요구하는 기본 기능
- **중간**: Figma 연동은 디자이너 워크플로우 핵심

### 요약
| 항목 | 완성도 | 개선도 |
|------|--------|--------|
| 4가지 내보내기 포맷 | ✅ 있음 | - |
| PDF 내보내기 | ❌ 없음 | 높음 |
| 공개 링크 | ❌ 없음 | 매우 높음 |
| 외부 공유 | ❌ 없음 | 매우 높음 |
| Figma 동기화 | ⚠️ 부분 | 높음 |

## 3. 협업 기능 완성도

### 댓글 시스템

#### 현재 구현: **기본 기능 완전** ✅
- **코드 근거**: `src/app/api/comments/route.ts`, `Comment` 모델
- **기능**:
  - 댓글 작성, 수정, 삭제 (소프트 삭제)
  - 답글 지원 (대댓글, 3단계 제한)
  - 해결 마킹 (`isResolved`, `resolvedById`)
  - 이메일 알림 (Brevo 통합)

#### 부족한 기능:
- **❌ @멘션**: 특정 팀원 언급 불가능
- **❌ 리액션**: 이모지 반응 없음
- **❌ 스레드**: 댓글 스레드화 UI 부족 (데이터는 있지만 UI 제한)
- **❌ 태깅**: 댓글에 우선순위/카테고리 태그 불가
- **❌ 검색**: 댓글 내용 검색은 있으나 필터링 없음

### 실시간 동시 편집

#### 현재 상태: **없음** ❌
- **문제**: 다중 사용자가 같은 생성 결과를 동시에 편집 불가 (편집 기능 자체가 부재)
- **영향**: 협업 흐름이 순차적 → 생산성 저하

### 알림 시스템

#### 현재 구현: **기본적인 인앱 알림** ✅

  - `comment.reply`: 댓글 답글
  - `generation.completed`: 생성 완료
  - `member.invited`: 팀원 초대
- **채널**: 인앱 알림 + 이메일 (댓글 답글만)

#### 부족한 기능:
- **❌ 푸시 알림**: 모바일/브라우저 푸시 없음
- **❌ Slack 통합**: Slack 채널로 알림 전송 불가
- **❌ 알림 필터/구독**: 사용자가 원하는 알림만 받을 수 없음
- **❌ 다이제스트**: 일일/주간 요약 알림 없음

### 팀원 초대 및 관리

#### 현재 구현: **완전** ✅
- **코드 근거**: `src/app/api/invitations/`, `Invitation` 모델
- **기능**:
  - 이메일 초대 (토큰 기반)
  - 역할 기반 (owner, admin, member)
  - 초대 만료 (48시간 기본값)
  - 수락/거절 플로우

#### 부족한 기능:
- **⚠️ 역할 세분화 부족**:
  - editor, viewer 등 세밀한 권한 없음
  - 모든 member는 동일 권한
  - 프로젝트 레벨 권한 제어 없음

## 5. 에이전트 시스템 완성도

### 에이전트 작업 관리

#### 현재 구현: **중간 수준** ⚠️
- **코드 근거**: `AgentTask`, `AgentConnection` 모델, `/api/agent/` 엔드포인트
- **기능**:
  - Self-hosted 에이전트 + Hosted 워커 (Fly.io) 지원
  - 작업 큐 관리 (우선순위 기반)
  - 작업 상태 추적 (pending → assigned → running → completed)
  - 타임아웃 설정 (기본 5분)
  - 에이전트별 작업 할당 (FOR UPDATE SKIP LOCKED)

#### 부족한 기능:
- **⚠️ 작업 일시중지/재개**: 실행 중인 작업 제어 불가능
- **⚠️ 작업 스케줄링**: 특정 시간에 자동 실행 불가 (즉시 실행만 가능)
- **❌ 작업 의존성**: 작업 간 순서 제어 불가능

### 에이전트 모니터링

#### 현재 구현: **기본 수준** ✅
- **코드 근거**: `/api/agent/dashboard`, `/api/admin/workers`
- **정보**:
  - 에이전트 상태 (online/offline)
  - 마지막 하트비트
  - 실행 중인 작업 수
  - 워커 풀 상태 (Fly.io)
  - 평균 처리 시간, 에러율

#### 부족한 기능:
- **❌ 상세 로그 UI**: 에이전트 수행 과정 상세 기록 표시 불가
- **❌ 성능 메트릭**: CPU, 메모리, 네트워크 사용량 모니터링 없음
- **❌ 알림**: 에이전트 실패 시 자동 알림 없음
- **❌ 메트릭 스토리지**: 시간별 성능 데이터 저장 안 됨

### 에이전트 로깅/디버깅

#### 현재 상태: **기본적** ⚠️
- **코드 근거**: `AgentTask.outputLog`, `AgentTask.errorMessage`
- **기능**:
  - 작업 출력 로그 저장
  - 에러 메시지 기록
- **부족한 점**:
  - **❌ 구조화된 로깅**: 플레인 텍스트 로그만
  - **❌ 로그 검색**: 로그 내용 검색 불가
  - **❌ 로그 레벨**: DEBUG, INFO, WARN, ERROR 구분 없음
  - **❌ 실시간 스트리밍**: 실행 중 로그 확인 불가

### 에이전트 확장성

#### 현재 상태: **높음** ✅
- **자동 스케일링**: Fly.io 호스티드 워커는 작업 큐 깊이에 따라 자동 확장
- **멀티 워커**: 여러 에이전트/워커 병렬 실행 가능
- **분산 작업**: 조직별 작업 분리

#### 제약사항:
- **⚠️ 워커 풀 상한**: 명시적 최대 워커 수 설정 불가능
- **❌ 리소스 제한**: 워커당 CPU/메모리 할당량 제어 불가
