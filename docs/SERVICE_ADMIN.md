## 1. 전체 사용자 플로우 개요

```
[지점 리스트]
    │
    ├── [+ 지점 생성] 클릭
    │       └── 생성 모달 Step 1: 지점 선택
    │               └── Step 2: 기본 정보 입력
    │                       └── [생성] → 지점 상세 (비공개 상태로 진입)
    │
    └── 지점 행 클릭
            └── [지점 상세]
                    │  ├── 상태 바: 홈페이지 공개 토글 / 노출 순서 / 탭 활성화
                    │  │
                    │  ├── [지점 소개 탭]
                    │  │       ├── 📋 기본 정보 모드 → 폼 편집 → [저장]
                    │  │       └── 📐 페이지 에디터 모드 → 블록 편집 → [저장]
                    │  │
                    │  ├── [공유오피스 탭] → 소개 이미지/텍스트, 블로그 후기 → [저장]
                    │  │
                    │  ├── [1인실 탭] → 탭 활성화, 상품 목록 관리 → [저장]
                    │  │
                    │  └── [라운지 멤버십 탭] → 탭 활성화, 이미지/CTA/좌석/서비스 → [저장]
                    │
                    └── [수정] 버튼 클릭
                            └── 수정 페이지: 공개 설정 / 기본 정보 / 탭 활성화 → [저장]
```

## 2. 플로우 A — 지점 생성

### 진입
지점 리스트 우측 상단 **[+ 지점 생성]** 버튼 클릭 → 모달 오픈

---

### Step 1 · 지점 선택

공간 어드민에 등록되어 있으나 **아직 서비스 어드민에 생성되지 않은 지점**만 목록에 노출된다. 이미 생성된 지점은 선택 불가 (1:1 매핑 원칙).

| 항목 | 입력 방식 | 필수 여부 | 비고 |
|------|-----------|-----------|------|
| 지점 선택 | 드롭다운 | 필수 | 미생성 지점만 노출 |
| 지점 유형 | 자동 입력 | — | 공간 어드민 기반, Read-only |
| 기본 주소 | 자동 입력 | — | 공간 어드민 기반, Read-only |
| 지역 분류 | 자동 입력 후 수정 가능 | 필수 | 기본값은 공간 어드민 기반 |

지점을 선택하면 지점 유형·기본 주소·지역 분류가 자동으로 채워진다. 지역 분류만 변경할 수 있다.

**[다음]** 클릭 → Step 2로 이동 (지점 미선택 시 진행 불가)

---

### Step 2 · 기본 정보 입력

| 항목 | 입력 방식 | 필수 여부 |
|------|-----------|-----------|
| 노출 지점명 | 텍스트 (최대 32자) | 필수 |
| 지상 층수 | 숫자 | 선택 |
| 지하 층수 | 숫자 (없으면 0) | 선택 |
| 지점 오픈일 | 날짜 선택 | 선택 |
| 인근 지하철역 | 드롭다운 | 선택 |
| 운영 담당자 | 드롭다운 | 선택 |

**[이전]** → Step 1로 복귀
**[생성]** → 모달 닫힘, 해당 지점 상세 페이지로 자동 이동

> ⚠️ 생성 직후 지점의 홈페이지 공개 여부는 **비공개** 상태로 시작된다.

## 3. 플로우 B — 지점 조회 및 탐색

### 지점 리스트 화면

전체 지점 목록을 테이블 형태로 표시한다.

**표시 컬럼**

| 컬럼 | 설명 |
|------|------|
| 지점 ID | 공간 어드민 고유 ID |
| 지점명 | 썸네일 + 노출 지점명 |
| 지역명 | 공간 어드민 지역 분류 |
| 인근 지하철역 | |
| 지점 유형 | 패스트파이브 / 파이브스팟 / 파워드바이 / 제휴 |
| 홈페이지 공개 | 공개 (초록) / 비공개 (회색) 배지 |
| 노출 순서 | 상위 / 기본 |

---

### 검색

검색창에 입력한 문자열이 지점명 또는 지역명 필드에 **완전히 포함**되는 경우에만 결과로 노출된다.

- 매칭 성공: `'대관'` 검색 → `'외부 대관'` 노출
- 매칭 실패: `'대관spot'` 검색 → `'외부 대관'` 제외 (일부 미포함)

검색 결과 내 일치 텍스트는 시각적으로 강조된다.

---

### 필터

| 필터 | 옵션 |
|------|------|
| 지역명 | 공간 어드민 지역 분류 체계 상속 (시/구 단위) |
| 지점 유형 | 패스트파이브 / 파이브스팟 / 파워드바이 / 제휴 |
| 홈페이지 공개 여부 | 공개 / 비공개 |

필터는 복합 적용된다. **[↺]** 클릭 시 전체 초기화.

---

### 지점 행 클릭

테이블의 임의 행 클릭 → 해당 지점의 **상세 페이지**로 이동

---

### 지점 삭제 정책

지점은 삭제하지 않는다. 운영 종료 시 홈페이지 공개를 **비공개로 전환**하여 관리한다.

## 4. 플로우 C — 지점 상세 관리

### 상세 페이지 상단 구조

**지점 헤더**
지점 썸네일, 지점명, ID·유형·지역을 표시한다. 우측에 현재 공개 상태 배지와 **[수정]** 버튼이 있다.

**상태 바**
각 토글과 배지를 통해 상세 페이지 진입 없이도 주요 상태를 즉시 변경할 수 있다.

| 컨트롤 | 역할 |
|--------|------|
| 홈페이지 공개 토글 | 공개 ↔ 비공개 즉시 전환 |
| 노출 순서 배지 | 상위 노출 / 기본 노출 표시 |
| 1인실 탭 토글 | 해당 탭의 홈페이지 노출 활성화 / 비활성화 |
| 라운지 멤버십 탭 토글 | 해당 탭의 홈페이지 노출 활성화 / 비활성화 |

**탭 메뉴**

```
[지점 소개]  [공유오피스]  [1인실]  [라운지 멤버십]
```

---

### 4-1. 지점 소개 탭 · 기본 정보 모드

탭 진입 후 기본 활성 모드. **[📋 기본 정보]** 버튼이 선택된 상태.

---

#### 섹션 1 · 기본 정보

공간 어드민 기반 데이터로, **서비스 어드민에서는 수정할 수 없다.**

| 항목 | 값 예시 |
|------|---------|
| 지점 ID | 13 |
| 지점 유형 | 패스트파이브 |
| 지역 분류 | 서울특별시 강남구 |
| 기본 주소 | 서울특별시 강남구 테헤란로 148 |

아래 항목은 서비스 어드민에서 직접 입력·수정한다.

| 항목 | 필수 여부 |
|------|-----------|
| 상세 주소 | 필수 |
| 위치 상세 안내 | 선택 |
| 인근 지하철역 안내 | 선택 |

---

#### 섹션 2 · 홈페이지 노출 정보

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 노출 지점명 | 필수 | 홈페이지에 표시될 지점명 |
| 한 줄 소개 | 선택 | 지점 소개 한 줄 문구 |
| 지점 뱃지 | 선택 | 신규오픈 / 운영예정 / 이벤트 / 인기 중 복수 선택 |
| 지점 이미지 | 선택 | 공간 어드민 기반. 추가 업로드 및 대표 이미지 설정 가능 |

---

#### 섹션 3 · 지점 소개 파일

지점 소개 PDF를 업로드한다. 업로드된 파일은 **공유오피스 탭의 '지점 소개 자료 다운로드' 버튼과 공용으로 사용**된다.

---

#### 섹션 4 · 시설 정보

공간 어드민 기반 수량이 기본값으로 채워지며, 관리자가 수정할 수 있다.

| 시설 | 기본값 예시 |
|------|------------|
| 주차 | 50대 |
| 폰부스 | 8개 |
| 미팅룸 | 12개 |
| 세미나룸 | 2개 |
| 라운지 | 1개 |
| 스튜디오 | 1개 |

입주사 전용 혜택 텍스트도 별도 입력할 수 있다.

---

#### 섹션 5 · 운영 시간

월요일~일요일/공휴일 각각의 시작 시간과 종료 시간을 개별 설정한다.

---

#### 저장

하단 **[초기화]** 또는 **[저장]** 버튼으로 완료한다.

### 4-2. 지점 소개 탭 · 페이지 에디터 모드

**[📐 페이지 에디터]** 버튼 클릭 시 전환. 마케터가 코딩 없이 홈페이지 지점 상세 페이지의 레이아웃과 콘텐츠를 직접 편집할 수 있다.

---

#### 에디터 기본 동작

| 동작 | 방법 |
|------|------|
| 블록 순서 변경 | 블록을 위아래로 드래그 앤 드롭 |
| 블록 내용 편집 | 블록 클릭 → 우측 편집 패널에서 텍스트·설정 수정 |
| 블록 복제 | 블록 툴바 **[⧉ 복제]** 클릭 |
| 블록 삭제 | 블록 툴바 **[🗑 삭제]** 클릭 |
| 블록 추가 | 하단 **[+ 블록 추가]** 클릭 → 블록 타입 선택 |
| 미리보기 | 상단 **[👁 미리보기]** 클릭 |
| 저장 | 상단 **[저장]** 클릭 → 즉시 홈페이지 반영 |

선택된 블록은 주황색 테두리로 강조된다. 드래그 중인 블록은 반투명 처리되고, 드롭 위치는 파란색 테두리로 피드백된다.

---

#### 지원 블록 타입 6종

**배너 블록**
큰 타이틀을 강조하는 다크 배경 배너.

편집 항목: 메인 타이틀, 서브 타이틀

---

**텍스트 블록**
섹션 제목과 본문으로 구성된 텍스트 영역.

편집 항목: 섹션 제목, 본문 텍스트

---

**이미지 블록**
1~3장의 이미지를 배치하는 블록.

편집 항목: 레이아웃 (와이드 1장 / 그리드), 이미지 수 (1 / 2 / 3장), 캡션

---

**이미지 + 텍스트 블록**
이미지와 텍스트를 좌우로 나란히 배치하는 블록.

편집 항목: 이미지 위치 (← 왼쪽 / 오른쪽 →), 섹션 제목, 본문 텍스트

---

**시설 목록 블록**
시설 항목을 태그 형태로 나열하는 블록.

편집 항목: 섹션 제목, 시설 항목 목록 (줄마다 하나씩 입력)

---

**버튼 블록**
CTA 버튼을 가운데 정렬로 배치하는 블록. 버튼 복수 추가 가능.

각 버튼 편집 항목: 버튼 문구, 랜딩 URL, 스타일 (채움 / 테두리)

---

#### 기본 블록 구성 (신규 지점 생성 시 초기값)

```
1. 배너     — 지점명 타이틀 + 위치 정보
2. 이미지   — 3장 그리드
3. 텍스트   — 지점 소개 제목 + 본문
4. 이미지 + 텍스트  — 공간 특징 소개
5. 시설 목록 — 주차·미팅룸·폰부스 등
6. 버튼     — 입주 문의하기 + 투어 신청
```

### 4-4. 1인실 탭

> ℹ️ [1인실] 메뉴에서 생성된 상품 중 **해당 지점에 존재하는 공간 유형**의 상품만 노출 대상이 된다.

---

#### 섹션 1 · 탭 설정

| 컨트롤 | 설명 |
|--------|------|
| 1인실 탭 노출 토글 | 활성화 시 홈페이지 지점 상세의 [1인실] 탭 노출 |

---

#### 섹션 2 · 1인실 상품 목록

현재 해당 지점에 연결된 1인실 상품을 리스트로 표시한다.

각 상품 카드에는 공간 유형 아이콘, 상품명, 공간 유형, 가격, 판매 상태 배지가 표시된다.

| 판매 상태 | 배지 색상 |
|-----------|-----------|
| 판매 중 | 초록 |
| 미판매 | 회색 |

## 6. 데이터 원칙 — Read-only vs 편집 가능

서비스 어드민의 지점 데이터는 두 계층으로 구분된다.

**공간 어드민 기반 (Read-only)**
원천 데이터는 공간 어드민이며, 서비스 어드민에서는 조회만 가능하다. 변경이 필요하면 공간 어드민에서 선행 수정 후 서비스 어드민에 자동 반영된다.

해당 항목: 지점 ID, 지점 유형, 기본 주소, 지역 분류, 인근 지하철역, 공간 면적·수용 인원·옵션, 기본 가격

**서비스 어드민 독자 관리 (편집 가능)**
홈페이지 노출을 위한 마케팅 정보는 서비스 어드민에서 독자적으로 설정한다.

해당 항목: 노출 지점명, 한 줄 소개, 지점 뱃지, 상세 주소, 위치 상세 안내, 소개 이미지·텍스트, 시설 수량, 운영 시간, 공개 여부, 노출 순서, 탭 활성화, CTA 버튼, 페이지 에디터 블록 구성

**공간 어드민 비활성화 연동**
공간 어드민에서 지점이 비활성화되면 서비스 어드민의 해당 지점 홈페이지 공개 여부가 자동으로 비공개로 전환된다.
